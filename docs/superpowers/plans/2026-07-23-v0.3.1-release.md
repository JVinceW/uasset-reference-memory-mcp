# v0.3.1 Documentation And Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish `unity-asset-reference-mcp@0.3.1` with the completed reindex reliability fixes and three supporting Markdown artifacts.

**Architecture:** Finish documentation on `fix/v0.3.1-reindex-reliability`, merge that reviewed branch into current `main`, and create one release commit that changes only the npm package version and lockfile. Publish through the existing `release/*` GitHub Actions workflow, verify the npm registry result, and only then create and push immutable tag `v0.3.1`.

**Tech Stack:** Markdown, Git, npm, GitHub Actions, PowerShell.

## Global Constraints

- Release version is exactly `0.3.1`, classified as PATCH by ADR 0012.
- `unity-asset-reference-mcp@0.3.1` and tag `v0.3.1` must not already exist.
- Preserve current `main` commit `1dc41c6` and its repo-local `.codex/config.toml` unchanged.
- Keep SQLite schema version `3`; do not alter public MCP/CLI input or successful response shapes.
- Do not bump `unity/com.jvincew.assetreferencememory/package.json`; the Unity verification package remains `0.2.0`.
- Do not implement analytics, watchers, Unity callbacks, or the deferred concurrent `.meta` read retry.
- Publish through `.github/workflows/release.yml`; do not run local `npm publish`.
- Do not create or push `v0.3.1` until npm confirms `unity-asset-reference-mcp@0.3.1`.
- Preserve unrelated worktree state and do not modify global Codex configuration.
