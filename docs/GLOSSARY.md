# Glossary

## Harness

The repository workflow that helps humans and agents turn intent into safe,
validated product changes.

## Product Contract

The current expected product behavior. Product docs and executable tests become
the living contract once implementation exists.

## Story Packet

A bounded work document that describes intended behavior, relevant context, and
validation expectations.

## Execution Plan

Git-tracked working memory for complex or multi-session work. Active plans live
under `docs/plans/active/`; validated plans move to `docs/plans/completed/`.

## Decision Record

A Git-tracked explanation of a lasting product, architecture, workflow, or
validation choice.

## Validation Evidence

Observed proof such as test output, integration results, runtime signals,
screenshots, logs, or reproducible recovery checks.
