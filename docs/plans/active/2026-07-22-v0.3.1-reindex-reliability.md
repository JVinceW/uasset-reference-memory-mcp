# v0.3.1 Reindex Reliability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make incremental reindexing follow Unity GUID identity across `.meta` edits, moves, replacements, duplicates, and incomplete pairs while preserving an explicit force-rebuild freshness path.

**Architecture:** Keep schema 3 and one row per Unity GUID. The scanner supplies one effective asset/`.meta` timestamp, index-boundary validation rejects ambiguous duplicate GUIDs, and incremental reconciliation matches by GUID before interpreting paths. Query tools remain read-only; agent refresh behavior is explicit guidance rather than hidden server automation.

**Tech Stack:** TypeScript 5.7, Node.js filesystem promises, SQLite through better-sqlite3, Vitest 2.1, MCP SDK 1.29.

## Global Constraints

- Release target is `0.3.1`; do not change `package.json` until a release is explicitly requested.
- Keep schema 3 and existing MCP/CLI argument and success-response shapes.
- Keep the asset and sibling `.meta` as one logical node and database row.
- Unity's GUID and `.meta` workflow is authoritative; never mutate or repair the Unity project.
- Query tools never index implicitly; add no watcher, timer, polling loop, or Unity callback.
- Case 15 and analytics JSON reporting are outside `0.3.1`; analytics is a separately designed `0.4.0` capability.
- Preserve the repository-local untracked `.codex/` directory and unrelated worktree state.

```ts
export interface GuidCollision { guid: string; paths: string[] }

export class DuplicateGuidError extends Error {
  constructor(public readonly collisions: GuidCollision[]) {
    super(formatDuplicateGuidMessage(collisions));
    this.name = "DuplicateGuidError";
  }
}

export function assertUniqueAssetGuids(
  nodes: readonly AssetNode[],
  reserved: readonly AssetNode[] = [],
): void {
  const pathsByGuid = new Map<string, string[]>();
  for (const node of [...reserved, ...nodes]) {
    const paths = pathsByGuid.get(node.guid) ?? [];
    paths.push(node.path);
    pathsByGuid.set(node.guid, paths);
  }
  const collisions = [...pathsByGuid]
    .filter(([, paths]) => paths.length > 1)
    .map(([guid, paths]) => ({ guid, paths: [...paths].sort() }))
    .sort((a, b) => a.guid.localeCompare(b.guid));
  if (collisions.length > 0) throw new DuplicateGuidError(collisions);
}
```

- Concurrent Unity/source-control activity can produce temporary incomplete-pair warnings. Rerun after stability; use force for guaranteed freshness.
- Each task is independently committed and revertible without migration because schema 3 remains unchanged.
