# Stories

Stories are work packets that turn product intent into bounded implementation
and validation work. The existing packets under `backlog.md` and its linked
story files are historical implementation evidence; create new packets only
when a change needs durable, story-sized coordination.

Use `docs/templates/story.md` for normal work and
`docs/templates/high-risk-story/` for work that needs separate design,
validation, or recovery artifacts.
